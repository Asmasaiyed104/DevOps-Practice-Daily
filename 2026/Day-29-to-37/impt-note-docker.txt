Why daemon off; Important?

Normally nginx runs in background.

But inside Docker:

main process must stay active

So:

daemon off;

keeps nginx running in foreground.

If main process stops:

container stops automatically

Very important Docker concept.

docker start starts an existing container that was already created or stopped earlier. Docker mounts the image layers, 
attaches networking, creates a writable layer, and starts the container's main process.
 In nginx container, Docker starts the nginx process using the CMD instruction from the image

docker stop performs graceful shutdown by sending SIGTERM signal, so the application gets time to clean up resources properly. But if the application becomes unresponsive or hangs, Docker may not be able to stop it normally.
 In that case we use docker kill, which sends SIGKILL and forcefully terminates the container immediately.
